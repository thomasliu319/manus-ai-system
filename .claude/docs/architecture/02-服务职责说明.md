## 2. 核心模块详解

### 2.1 pipeline/pipeline.py — 流水线主编排

**职责**：编排四步流水线（采集 → 分析 → 整理 → 保存），提供 CLI 入口。

**文件大小**：~330 行

**对外接口**：

| 接口 | 类型 | 说明 |
|------|------|------|
| `run_pipeline(sources, limit, dry_run, steps)` | 函数 | 编程式调用入口 |
| `main()` → CLI argparse | CLI | 命令行入口 |

**CLI 参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--sources` | str | `github,rss` | 逗号分隔的数据源 |
| `--limit` | int | 20 | 每个源最大采集数 |
| `--dry-run` | flag | False | 模拟运行不写文件 |
| `--verbose` | flag | False | 详细日志 |
| `--step` | int (可多选) | 全部 | 指定执行步骤 1-4 |
| `--provider` | str | 环境变量 | LLM 提供商覆盖 |

**依赖**：
- `model_client.py` → `create_provider()`, `chat_with_retry()`, `estimate_cost()`
- `rss_reader.py` → `collect_rss()`
- `httpx` → GitHub API 请求
- `yaml` → RSS 配置读取
- `dotenv` → 环境变量加载

**代码位置**：`v2-automation/pipeline/pipeline.py`

---

### 2.2 pipeline/model_client.py — LLM 客户端

**职责**：统一封装 DeepSeek/Qwen/OpenAI 三家 LLM 的调用，提供工厂创建、重试、成本估算。

**文件大小**：~260 行

**核心类/函数**：

| 名称 | 类型 | 说明 |
|------|------|------|
| `Usage` | dataclass | Token 用量统计（prompt + completion） |
| `LLMResponse` | dataclass | 统一响应格式（content + usage） |
| `LLMProvider` | ABC | 提供商抽象基类 |
| `OpenAICompatibleProvider` | class | OpenAI 兼容实现 |
| `create_provider(name)` | 工厂函数 | 创建 LLM 客户端 |
| `chat_with_retry(...)` | 函数 | 带指数退避重试的调用 |
| `estimate_cost(model, usage)` | 函数 | 估算调用成本 |
| `chat(prompt, ...)` | 便捷函数 | 一行调用 LLM |

**支持的模型定价**：

| 模型 | Input/1K | Output/1K |
|------|----------|-----------|
| deepseek-v4-pro | $0.0014 | $0.0028 |
| deepseek-v4-flash | $0.004 | $0.016 |
| qwen-plus | $0.002 | $0.006 |
| qwen-turbo | $0.0005 | $0.001 |
| gpt-4o-mini | $0.00015 | $0.0006 |
| gpt-4o | $0.005 | $0.015 |

**代码位置**：`v2-automation/pipeline/model_client.py`

---

### 2.3 pipeline/rss_reader.py — RSS 采集器

**职责**：从配置的 RSS 源采集 AI 技术内容，返回统一格式的原始数据。

**文件大小**：~85 行

**对外接口**：

| 接口 | 说明 |
|------|------|
| `collect_rss(limit=10)` | 采集 RSS 内容，返回 `list[dict]` |

**数据流**：
1. 读取 `rss_sources.yaml` 配置
2. 过滤 `enabled: true` 的源
3. HTTP GET 获取 RSS XML
4. 正则解析 `<item>` → `<title>` + `<link>`
5. 返回统一格式（id/title/source/source_url/...）

**代码位置**：`v2-automation/pipeline/rss_reader.py`

---

### 2.4 mcp_knowledge_server.py — MCP 知识检索服务

**职责**：通过 MCP 协议（JSON-RPC 2.0 over stdio）暴露知识库检索能力。

**文件大小**：~165 行

**提供的 3 个 MCP 工具**：

| 工具 | 参数 | 说明 |
|------|------|------|
| `search_articles` | keyword (必填), limit (默认 5) | 按关键词搜索标题/摘要/标签 |
| `get_article` | article_id (必填) | 按 ID 获取完整文章内容 |
| `knowledge_stats` | 无 | 统计信息（总数、来源分布、热门标签） |

**协议实现**：JSON-RPC 2.0
- `initialize` — 服务初始化握手
- `tools/list` — 返回工具列表和输入模式
- `tools/call` — 调用具体工具
- `notifications/initialized` — 初始化完成通知

**代码位置**：`v2-automation/mcp_knowledge_server.py`

---

### 2.5 hooks/validate_json.py — JSON 校验工具

**职责**：校验知识条目 JSON 文件的格式正确性。

**校验项**：

| 校验项 | 规则 |
|--------|------|
| 必填字段 | id, title, source_url, summary, tags, status |
| ID 格式 | `^[a-z][a-z0-9-]*-\d{8}-\d{3}$` |
| URL 格式 | `^https?://\S+$` |
| Status 枚举 | draft/review/published/archived |
| Score 范围 | 1-10 |
| Audience 枚举 | beginner/intermediate/advanced |
| Summary 长度 | ≥ 20 字符 |
| Tags 非空 | 至少 1 个标签 |

**退出码**：0 = 全部通过，1 = 存在失败

**代码位置**：`v2-automation/hooks/validate_json.py`

---

### 2.6 hooks/check_quality.py — 质量评分工具

**职责**：5 维度质量评分，输出 A/B/C 等级。

**5 维度**：

| 维度 | 满分 | 评分逻辑 |
|------|------|----------|
| 摘要质量 | 25 | 长度 + 技术关键词密度 |
| 技术深度 | 25 | `score 字段值 × 2.5` |
| 格式规范 | 20 | 5 项字段格式检查（各 4 分） |
| 标签精度 | 15 | 标签数量 + 是否在标准白名单中 |
| 空洞词检测 | 15 | 每发现一个空洞词扣 5 分 |

**等级**：A (≥80), B (≥60), C (<60)

**退出码**：0 = 无 C 级，1 = 存在 C 级

**代码位置**：`v2-automation/hooks/check_quality.py`

## 3. Agent 角色定义

### collector.md — 数据采集员

- **权限**：Read, Grep, Glob, WebFetch（**禁止 Write**）
- **数据源**：GitHub Trending, Hacker News
- **输出**：`knowledge/raw/{source}-{date}.json`
- **代码位置**：`.opencode/agents/collector.md`

### analyzer.md — 深度分析员

- **权限**：Read, Grep, Glob, WebFetch（**禁止 Write**）
- **输入**：`knowledge/raw/` 中的原始数据
- **输出**：摘要（100-200 字中文）+ 5 维评分 + 标签
- **代码位置**：`.opencode/agents/analyzer.md`

### organizer.md — 整理归档员

- **权限**：Read, Grep, Glob, Write, Edit（**禁止 WebFetch**）
- **输入**：分析后的数据
- **输出**：标准化知识条目 + `index.json` 维护
- **质量**：`relevance_score < 0.6` 丢弃
- **代码位置**：`.opencode/agents/organizer.md`

```
┌─────────────────────────────────────────────────────────────┐

│  ├── pipeline/model_client.py  (LLM 调用)                   │

│  ├── pipeline/rss_reader.py  (RSS 采集)                     │
│  │   ├── httpx                                              │
│  │   └── pipeline/rss_sources.yaml                          │
│  ├── python-dotenv  (.env 加载)                             │
│  └── yaml  (配置解析)                                       │
├─────────────────────────────────────────────────────────────┤
│  hooks/validate_json.py   (独立工具，CLI 调用)               │
│  hooks/check_quality.py   (独立工具，CLI 调用)               │
│  mcp_knowledge_server.py  (独立服务，stdio 运行)             │
├─────────────────────────────────────────────────────────────┤
│  .opencode/agents/        (OpenCode IDE 调用)               │
│  .opencode/skills/        (OpenCode IDE 调用)               │
└─────────────────────────────────────────────────────────────┘
```

**关键依赖关系**：
- `pipeline.py` → `model_client.py`：调用 LLM 分析
- `pipeline.py` → `rss_reader.py`：采集 RSS 数据
- `model_client.py` → `httpx`：HTTP API 请求
- `mcp_knowledge_server.py` → `knowledge/articles/`：读取本地知识条目
- `hooks/*` → `knowledge/articles/`：校验/评分知识条目
- `daily-collect.yml` → `pipeline.py` → `hooks/*`：CI/CD 全链路
