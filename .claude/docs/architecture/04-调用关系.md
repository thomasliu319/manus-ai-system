## 2. LangGraph 工作流调用链（v3）

### 2.1 状态图构建

```
build_graph()                                      # graph.py:27
  ├── StateGraph(KBState)                          # 绑定共享状态类型
  ├── .add_node("collect", collect_node)
  ├── .add_node("analyze", analyze_node)
  ├── .add_node("review", review_node)
  ├── .add_node("organize", organize_node)
  ├── .add_node("save", save_node)
  ├── .add_edge(START, "collect")
  ├── .add_edge("collect", "analyze")
  ├── .add_edge("analyze", "review")
  ├── .add_edge("review", "organize")
  ├── .add_conditional_edges("organize", _organize_router,
  │       {"save": "save", "review": "review"})
  ├── .add_edge("save", END)
  └── .compile()
```

### 2.2 运行时执行流

```
app.invoke(initial_state)
  │
  ├── collect_node(state)                          # nodes.py
  │     └── 返回 {"sources": [...], "cost_tracker": {...}}
  │
  ├── analyze_node(state)                          # nodes.py
  │     ├── 读取 state["sources"]
  │     ├── for each source:
  │     │     ├── 构建 LLM prompt
  │     │     ├── chat_json(prompt, system=_ANALYZE_SYSTEM)
  │     │     └── 追加到 analyses 列表
  │     └── 返回 {"analyses": [...], "cost_tracker": {...}}
  │
  ├── review_node(state)                           # reviewer.py
  │     ├── 取 state["analyses"][:5]
  │     ├── chat_json(REVIEW_PROMPT, system=_REVIEW_SYSTEM)
  │     ├── 获取 LLM 返回的 scores (5 维度)
  │     ├── 代码重算加权总分:
  │     │     weighted = sum(
  │     │         scores[dim] * _WEIGHTS[dim]
  │     │         for dim in _WEIGHTS
  │     │     )
  │     ├── passed = weighted >= _PASS_THRESHOLD (7.0)
  │     └── 返回 {"review_passed": bool, "review_feedback": str, ...}
  │
  ├── organize_node(state)                         # nodes.py
  │     ├── 读取 state["analyses"]
  │     ├── 去重 (source_url)
  │     ├── 格式标准化
  │     └── 返回 {"articles": [...], ...}
  │
  ├── [条件边] _organize_router(state)
  │     ├── state["review_passed"] == True  → "save"
  │     └── state["review_passed"] == False → "review" (循环)
  │
  └── save_node(state)                             # nodes.py
        ├── 读取 state["articles"]
        ├── for each article:
        │     └── json.dump → knowledge/articles/{id}.json
        └── 返回 {"cost_tracker": {...}}
```

## 3. MCP Server 请求处理链


```python
    "id": "github-20260704-001",          # str, 唯一标识

    "source": "github",                  # str, 来源标识
    "source_url": "https://...",         # str, 原始 URL
    "author": "langgenius",              # str, 作者/组织
    "published_at": "2026-07-04T...",    # str, ISO 8601
```

## 6. Router 模式调用链（v3）

```
用户输入 (query_str)
  │
  ├── classify_intent(query_str)                     # router.py
  │     │
  │     ├── 第一层: 关键词匹配
  │     │     └── 匹配成功 → 返回 intent
  │     │
  │     └── 第二层: LLM 分类 (兜底)
  │           ├── chat_json(INTENT_PROMPT)
  │           └── LLM 返回 {"intent": "github_search"|"knowledge_query"|"general_chat"}
  │
  └── dispatch(intent, query_str)                    # 按意图分发
        │
        ├── github_search
        │     └── search_github(query) → GitHub Search API → list[dict]
        │
        ├── knowledge_query
        │     └── search_local(query) → glob articles/*.json → 标题/摘要匹配
        │
        └── general_chat
              └── chat(query) → LLM 直接回答
```

## 7. Supervisor 模式调用链（v3）

```
supervise_task(task, context)                        # supervisor.py
  │
  └── for iteration in 1..3:
        │
        ├── Worker: generate_report(task, feedback)
        │     ├── chat_json(WORKER_PROMPT)
        │     └── return {"title": ..., "analysis": ...}
        │
        ├── Supervisor: review(report)
        │     ├── chat_json(SUPERVISOR_PROMPT)
        │     ├── LLM 返回 {"accuracy": 8, "depth": 7, "format": 9}
        │     └── total = accuracy + depth + format
        │
        ├── if total >= 21 (avg >= 7):  # 通过阈值
        │     └── return report
        │
        ├── else:  # 不通过
        │     ├── feedback = supervisor_output["feedback"]
        │     └── 继续下一轮 (带反馈)
        │
        └── 循环结束 (超过 3 轮):
              └── logger.warning + 强制返回当前结果
```
