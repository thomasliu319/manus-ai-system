# 04 - 异常处理规范

> 本文档汇总项目中的异常处理策略、错误恢复机制和各模块的具体实践。

---

## 1. 总体原则

| 原则 | 说明 |
|------|------|
| **不中断整体流程** | 单条数据处理失败不应导致整个流水线崩溃 |
| **记录错误** | 所有异常必须通过 `logger` 记录，关键错误写入持久化日志文件 |
| **降级兜底** | 当最优路径不可用时，使用次优默认值而非报错退出 |
| **可回溯** | 异常信息需包含上下文（数据 ID、处理阶段、失败原因） |

### 2.4 LLM 调用 — 指数退避重试

**代码位置**：`v2-automation/pipeline/model_client.py:199-231`

```python
def chat_with_retry(provider, messages, ..., max_retries=3, backoff_base=2.0):
    last_error = None
    for attempt in range(max_retries):
        try:
            response = provider.chat(...)
            if attempt > 0:
                logger.info("第 %d 次重试成功", attempt)
            return response
        except (httpx.HTTPStatusError, httpx.ConnectError, httpx.TimeoutException) as e:
            last_error = e
            if attempt < max_retries - 1:
                wait_time = backoff_base ** attempt  # 1s, 2s, 4s
                logger.warning("LLM 调用失败（第 %d/%d 次），%0.1f 秒后重试: %s",
                               attempt + 1, max_retries, wait_time, str(e))
                time.sleep(wait_time)
            else:
                logger.error("LLM 调用失败，已达最大重试次数: %s", str(e))
    raise last_error  # 所有重试失败后抛出
```

**策略**：最多重试 3 次，指数退避等待（1s → 2s → 4s），全部失败后向上抛出。

```python
def load_articles():
    articles = []
    if not ARTICLES_DIR.exists():
        return articles  # 目录不存在时返回空列表
    for f in sorted(ARTICLES_DIR.glob("*.json")):
        try:
            with open(f, "r", encoding="utf-8") as fh:
                articles.append(json.load(fh))
        except (json.JSONDecodeError, OSError):
            continue  # 损坏的文件静默跳过
    return articles
```

**策略**：损坏的 JSON 文件静默跳过，不中断整个加载流程。

### 4.1 CLI 退出码

| 退出码 | 含义 |

| 0 | 全部校验通过 |

### 4.2 条目状态流转

```
draft → review → published → archived
  ↑                 │
  └─────────────────┘ (审核不通过，回退修改)
```

| 状态 | 含义 |
|------|------|
| `draft` | 初始草稿（如 LLM 解析失败时的兜底状态） |
| `review` | 待审核（LLM 分析完成后的默认状态） |
| `published` | 已发布 |
| `archived` | 已归档 |

## 5. 异常处理反模式（禁止）

| 禁止 | 说明 | 正确做法 |
|------|------|----------|
| `except Exception` 裸捕获 | 掩盖意外错误 | 捕获具体的异常类型 |
| `except: pass` 无日志 | 丢失排查线索 | 至少记录 `logger.warning` |
| 异常后不设默认值 | 后续代码引用 `None` 崩溃 | 提供合理兜底值 |
| 重试无上限 | 可能导致死循环 | 最多 3 次重试 |
| 异常后不释放资源 | 连接泄漏 | 使用 `try/finally` 或上下文管理器 |
