## 1. 日志配置

### 1.1 默认配置

**代码位置**：`v2-automation/pipeline/pipeline.py:320-324`

```python
logging.basicConfig(
    level=logging.DEBUG if args.verbose else logging.INFO,
    format="%(asctime)s %(levelname)s %(name)s: %(message)s",
    datefmt="%H:%M:%S",
```

| 级别 | 触发条件 |
|------|----------|
| `DEBUG` | `--verbose` 标志开启时 |
| `INFO` | 默认级别 |

## 2. 各级别使用规范

### 2.1 INFO — 正常流程节点

用于记录每个阶段完成情况：

```python
logger.info("GitHub 采集完成: %d 条", len(results))    # pipeline.py:65
logger.info("RSS [%s] 采集: %d 条", source["name"], len(items))  # rss_reader.py:70
logger.info("创建 LLM 客户端: provider=%s, model=%s", name, model)  # model_client.py:187
```

### 2.2 WARNING — 可恢复错误

用于非致命问题，流程可以继续：

```python
logger.warning("分析结果解析失败: %s — %s", item["title"], e)  # pipeline.py:149
logger.warning("RSS 源 [%s] 获取失败: %s", source["name"], e)  # rss_reader.py:73
logger.warning("RSS 配置文件不存在: %s", RSS_CONFIG)           # rss_reader.py:38
logger.warning("LLM 调用失败（第 %d/%d 次），%0.1f 秒后重试: %s",
               attempt + 1, max_retries, wait_time, str(e))   # model_client.py:219
```

### 2.3 ERROR — 不可恢复错误

用于严重问题，需要人工关注：

```python
logger.error("GitHub API 调用失败: %s", e)                    # pipeline.py:67
logger.error("LLM 调用失败，已达最大重试次数: %s", str(e))    # model_client.py:224
```

## 3. 打印输出（print）使用规范

### 3.1 阶段标题

用于向用户展示进度，使用 `print` 而非 `logger`（因为它们是面向用户的，非开发者日志）：

```python
print(f"\n{'='*60}")
print(f"📥 Step 1: 采集（sources={sources}, limit={limit}）")
print(f"{'='*60}")
```

### 3.2 进度指示

```python
print(f"  [{i+1}/{len(items)}] 分析: {item['title'][:50]}...")
```

### 3.3 统计汇总

```python
print(f"  采集到 {len(all_items)} 条原始数据")
print(f"  保存到 {raw_file}")
print(f"  估算总成本: ${total_cost:.6f}")
```

### 3.4 流水线结果横幅

```python
print(f"\n{'#'*60}")
print(f"# 流水线完成！耗时 {elapsed:.1f} 秒")
print(f"# 采集: {stats['collected']} → 分析: {stats['analyzed']} "
      f"→ 整理: {stats['organized']} → 保存: {stats['saved']}")
print(f"{'#'*60}\n")
```

### 3.5 进度条（stderr）

**代码位置**：`v2-automation/hooks/check_quality.py:221-226`

```python
def print_progress(current: int, total: int, bar_width: int = 30):
    fraction = current / total if total > 0 else 0
    filled = int(bar_width * fraction)
    bar = "█" * filled + "░" * (bar_width - filled)
    print(f"\r  进度: [{bar}] {fraction:>4.0%} ({current}/{total})", end="",
          file=sys.stderr)
```

进度条输出到 `stderr`，避免与主输出的格式化结果混淆。

## 4. 日志与用户体验分离原则

| 通道 | 目标受众 | 内容 |
|------|----------|------|
| `print()` | 终端用户 | 阶段标题、进度、统计、结果 |
| `logger.info()` | 开发者/运维 | 组件操作成功（采集N条、模型初始化） |
| `logger.warning()` | 开发者/运维 | 可恢复错误（重试中、解析失败已兜底） |
| `logger.error()` | 开发者/运维 | 严重错误（API 无法访问、重试全部失败） |

## 6. 日志反模式（禁止）

| 禁止 | 原因 | 正确做法 |
|------|------|----------|
| `print()` 替代 logger | 丢失日志级别和时间戳信息 | 用 `logger.info()` 记录运行状态 |
| `logger.info()` 替代 print | 用户看不到进度 | 用 `print()` 输出用户可见的进度 |
| 无格式占位符的字符串拼接 | 性能浪费（即使不输出也会计算） | 用 `%s` 占位符延迟格式化 |
| `except: pass` 无日志 | 无法排查 | 至少 `logger.warning()` |
