# 02 - 命名规范

> 本文档汇总项目中的命名规范，涵盖文件、变量、JSON 字段、数据 ID 等方面。

---

## 1. 文件命名

### 1.1 知识数据文件

| 用途 | 格式 | 说明 | 示例 |
|------|------|------|------|
| 原始采集数据 | `raw_{YYYYMMDD}_{HHmmss}.json` | 时间戳精确到秒 | `raw_20260704_165313.json` |
| 知识条目 | `{source}-{YYYYMMDD}-{NNN}.json` | source: github/rss; NNN: 三位序号 | `github-20260704-001.json` |
| 已过滤条目 | `filtered-{YYYY-MM-DD}.json` | 记录被丢弃的条目及原因 | `filtered-2026-07-04.json` |
| 错误日志 | `errors-{date}.json` | API 调用异常记录 | `errors-2026-07-04.json` |
| 索引文件 | `index.json` | 固定名称 | `index.json` |

### 1.2 Python 模块

| 类别 | 规范 | 示例 |
|------|------|------|
| 模块文件 | `snake_case.py` | `model_client.py`, `rss_reader.py` |
| 包初始化 | `__init__.py` | `pipeline/__init__.py` |
| 配置文件 | `snake_case.yaml` | `rss_sources.yaml` |
| 环境变量模板 | `.env.example` | `.env.example` |

### 1.3 Agent 和 Skill 文件

| 类别 | 命名 |
|------|------|
| Agent 角色定义 | `.opencode/agents/{role}.md` — `collector.md`, `analyzer.md`, `organizer.md` |
| 技能定义 | `.opencode/skills/{skill-name}/SKILL.md` — `github-trending/SKILL.md` |
| 插件定义 | `.opencode/plugins/{plugin-name}.js` — `validate-hook.js` |

## 2. 变量命名

| 类别 | 规范 | 示例 |
|------|------|------|
| 模块级常量 | `UPPER_CASE` | `RAW_DIR`, `ARTICLES_DIR`, `MAX_ANALYSES` |

## 3. 数据 ID 命名

### 3.1 知识条目 ID

格式：`{source}-{YYYYMMDD}-{NNN}`

- `source`：小写字母，数据来源标识（`github` / `rss`）
- `YYYYMMDD`：采集日期（8 位数字）
- `NNN`：三位序号，从 001 开始，当天内递增

**代码位置**：`v2-automation/pipeline/pipeline.py:43` — ID 生成逻辑
```python
"id": f"github-{datetime.now().strftime('%Y%m%d')}-{i+1:03d}",
```

**校验正则**：`^[a-z][a-z0-9-]*-\d{8}-\d{3}$`
**代码位置**：`v2-automation/hooks/validate_json.py:21`

### 3.2 RSS 条目 ID

格式：`rss-{YYYYMMDD}-{NNN}`（与 GitHub 使用相同模板但 source 不同）
**代码位置**：`v2-automation/pipeline/rss_reader.py:60`

### 3.3 状态枚举值

| 字段 | 可选值 | 说明 |
|------|--------|------|
| `status` | `draft`, `review`, `published`, `archived` | 知识条目状态流转 |
| `audience` | `beginner`, `intermediate`, `advanced` | 目标读者技术水平 |

**代码位置**：`v2-automation/hooks/validate_json.py:16-17`

## 5. 代码注释规范

### 5.1 文件头 docstring

每个核心模块必须包含文件级 docstring，说明模块用途：

```python
"""
AI 知识库四步流水线：采集 → 分析 → 整理 → 保存

运行方式：
    python3 pipeline/pipeline.py --sources github,rss --limit 20
    python3 pipeline/pipeline.py --sources github --limit 5 --dry-run
"""
```

### 5.2 分段标题

使用 `# ── 区块名 ──` 格式分隔代码区块：

```python
# ── Step 1: 采集（Collect） ──────────────────────────────────────────────
```

### 5.3 函数文档

使用 Google 风格 docstring，包含 Args/Returns/Raises：

```python
def chat_with_retry(
    provider: LLMProvider,
    messages: list[dict[str, str]],
    temperature: float = 0.7,
    max_tokens: int = 2000,
    max_retries: int = 3,
    backoff_base: float = 2.0,
) -> LLMResponse:
    """
    带指数退避重试的聊天调用。

    Args:
        provider: LLM 提供商实例
        messages: 消息列表
        ...

    Returns:
        LLMResponse 统一响应

    Raises:
        最后一次重试仍失败时抛出原始异常
    """
```

## 6. 禁止的命名方式

| 禁止 | 原因 | 正确做法 |
|------|------|----------|
| 中文变量名 / 拼音 | 可读性差，不符合 PEP 8 | 使用英文单词 |
| 单字母变量（非循环变量） | 含义不清 | 使用描述性名称 |
| 缩写（未广泛认可的） | 增加理解成本 | 使用全称，如 `completion_tokens` 而非 `comp_tok` |
| `_` 前缀的公共 API | 混淆公私界限 | 仅私有方法/函数加 `_` 前缀 |
| camelCase 变量名 | 不符合 PEP 8 | 使用 snake_case |
