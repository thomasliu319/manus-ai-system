# 01 - 代码组织结构

> **项目名称**：Manus AI 知识库系统（AI Knowledge Base）
> **项目类型**：Python 单体应用 + AI Agent 驱动流水线
> **代码组织特点**：三版本渐进式演进 + OpenCode Agent 协作 + 四步流水线管道

---

## 1. 项目概览

Manus AI 知识库系统是一个**自动化技术情报采集与分析系统**，持续追踪 GitHub Trending、Hacker News、arXiv 等技术来源，将分散的技术资讯转化为结构化、可检索的知识条目。

### 核心价值

- 每日自动采集 AI/LLM/Agent 领域的高质量技术文章与开源项目
- 通过 Agent 协作完成 **采集 → 分析 → 整理** 三阶段流水线
- 输出格式统一的 JSON 知识条目，便于下游应用消费

---

## 2. 版本演进路线

| 版本 | 目录 | 定位 | 核心模式 |
|------|------|------|----------|
| **v1-skeleton** | `v1-skeleton/` | 骨架版 — 手动 Agent 驱动 | OpenCode `@agent` 调用，人工触发三阶段流水线 |
| **v2-automation** | `v2-automation/` | 自动化版 — 脚本驱动 | Python CLI 全自动流水线 + MCP 知识检索 |
| **v3-multi-agent** | `v3-multi-agent/` | 多智能体版 — 图驱动 | LangGraph 状态图 + 审核循环 + 多模式设计 |

### v1 → v2 的关键变化

- 从"手动 @agent"升级为 `python pipeline/pipeline.py --sources github,rss --limit 20`
- 引入 `model_client.py` 工厂模式，统一封装 DeepSeek/Qwen/OpenAI 调用
- 引入 `mcp_knowledge_server.py`，通过 MCP 协议暴露知识库检索能力
- 引入 GitHub Actions 每日定时自动采集 (`daily-collect.yml`)

### v2 → v3 的关键变化

- 从过程式 Python 脚本升级为 **LangGraph 状态图** (`workflows/graph.py`)
- 引入 **审核循环**（review loop）：分析 → 审核 → 不通过则重做，最多 3 轮
- 引入 **设计模式库** (`patterns/`)：Router 路由模式 + Supervisor 监督模式
- 引入 **报告式通信原则**：工作流节点间通过结构化状态字段通信

## 5. v3 新增模块详解

### 5.1 workflows/ — LangGraph 工作流

**graph.py** — 工作流拓扑：
```
START → collect → analyze → review → organize ─(passed)→ save → END
                                       ↑         │
                                       └─────────┘ (not passed)
```

- `build_graph()` 组装 StateGraph 并编译
- 条件边 `_organize_router`：通过 → save，不通过 → 回 review
- 代码位置：`v3-multi-agent/workflows/graph.py:27-58`

**state.py** — 共享状态 KBState (TypedDict)：
- `sources`: 采集的原始数据摘要列表
- `analyses`: LLM 分析后的结构化结果
- `articles`: 去重格式化后的最终知识条目
- `review_feedback`: 审核反馈文本
- `review_passed`: 审核是否通过（布尔值）
- `iteration`: 当前审核循环次数（1-based，上限 3）
- `cost_tracker`: Token 用量与成本累积追踪
- 代码位置：`v3-multi-agent/workflows/state.py:14-79`

**nodes.py** — 5 个节点函数：
| 节点 | 函数 | 输入 | 输出 |
|------|------|------|------|
| collect | `collect_node()` | GitHub API | sources 列表 |
| analyze | `analyze_node()` | sources | analyses（含 LLM 摘要/评分/标签） |
| organize | `organize_node()` | analyses | articles（去重 + 格式化） |
| review | `review_node()` | analyses | review_passed + feedback |
| save | `save_node()` | articles | knowledge/articles/*.json |
- 代码位置：`v3-multi-agent/workflows/nodes.py`

**reviewer.py** — 5 维度加权审核：
| 维度 | 权重 | 满分 |
|------|------|------|
| summary_quality | 0.25 | 10 |
| technical_depth | 0.25 | 10 |
| relevance | 0.20 | 10 |
| originality | 0.15 | 10 |
| formatting | 0.15 | 10 |
- 通过阈值：总分 >= 7.0
- 代码位置：`v3-multi-agent/workflows/reviewer.py:15-23`

### 5.2 patterns/ — 设计模式库

**router.py** — Router 路由模式：
- 第一层：关键词快速匹配（零成本，不调 LLM）
- 第二层：LLM 分类兜底（处理模糊意图）
- 三种意图分发：github_search / knowledge_query / general_chat
- 代码位置：`v3-multi-agent/patterns/router.py`

**supervisor.py** — Supervisor 监督模式：
- Worker → Supervisor 审核循环
- 通过 (score >= 7) → 返回结果
- 不通过 → 带反馈重做（最多 3 轮）
- 超过 3 轮 → 强制返回 + 警告
- 代码位置：`v3-multi-agent/patterns/supervisor.py`

## 6. 文件命名规范

| 文件类型 | 命名格式 | 示例 |

| 文件 | 命名规范 |
|------|----------|
| 模块文件 | `snake_case.py` |
