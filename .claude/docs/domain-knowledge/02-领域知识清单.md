## 2. 代码组织领域知识

### 2.1 版本演进策略

| 知识点 | 说明 |
|--------|------|
| **三版本并行维护** | v1/v2/v3 三个独立目录并存，v3 向后兼容 v2 的 pipeline/hooks 模块 |
| **渐进式复杂度** | v1(手动Agent) → v2(脚本自动化) → v3(图驱动+审核循环) |
| **代码复用** | v3 直接导入 v2 的 `pipeline/model_client.py`（patterns 模块），同时有自己的轻量版 `workflows/model_client.py` |

### 2.2 关键设计决策

| 决策 | 原因 | 影响 |
|------|------|------|
| 不使用第三方 RSS 库 | 减少依赖，控制复杂度 | 正则解析对非标准 RSS 可能存在兼容问题 |
| LLM 兜底默认值 (score=5) | 确保流水线不因单条 LLM 失败而中断 | 可能引入低质量条目 |
| 文件系统作为数据总线 | 简单可靠，无需数据库 | 并发写入安全依靠文件系统原子性 |
| MCP over stdio | 符合 MCP 标准，无需网络端口 | 与 OpenCode IDE 深度集成 |
| v3 使用 LangGraph 而非自建状态机 | 利用成熟的图执行引擎 | 引入新依赖 |
| 审核最多 3 轮 | 防止无限循环消耗 API 费用 | 可能遗漏需要深度改进的条目 |

### 2.3 需要特别理解的地方

1. **v2 和 v3 的 model_client.py 是两个独立文件**
   - v2: `pipeline/model_client.py` — 完整版，含工厂模式、定价表、便捷函数
   - v3: `workflows/model_client.py` — 轻量版，含 `chat()` 和 `chat_json()`，以及 `accumulate_usage()` 成本追踪辅助函数
   - patterns 模块使用的是 v2 版本

2. **去重发生在 organize 阶段，而非 collect 阶段**
   - 设计原因：保留完整的采集日志（raw/），只在输出时去重
   - 去重 key：`source_url`（而非 title 或 id）

3. **v3 审核只取前 5 条分析结果**
   - `reviewer.py:18`: `_MAX_ANALYSES = 5`
   - 设计原因：控制审核的 LLM 调用成本

4. **review 节点使用的是 1-10 的"伪分"再代码加权**
   - LLM 返回每维度 1-10 分
   - Python 代码用预定义权重重算加权总分
   - 通过阈值：总分 >= 7.0
   - 代码位置：`reviewer.py:15-23`

## 4. 常见修改场景

### 4.1 添加新的 RSS 源

**影响文件**：`pipeline/rss_sources.yaml`

```yaml
- name: "新源名称"
  url: "https://example.com/rss.xml"
  category: "research"  # general/research/open-source/industry/framework/chinese
  enabled: true
```

### 4.2 添加新的 LLM 提供商

**影响文件**：`pipeline/model_client.py`

1. 在 `PROVIDER_CONFIG` 字典中添加提供商配置
2. 在 `PRICING` 字典中添加模型定价（可选）
3. 无需修改 `create_provider()` 工厂函数

### 4.3 修改审核维度权重（v3）

**影响文件**：`v3-multi-agent/workflows/reviewer.py`

修改 `_WEIGHTS` 字典：
```python
_WEIGHTS: dict[str, float] = {
    "summary_quality": 0.25,
    "technical_depth": 0.25,
    "relevance": 0.20,
    "originality": 0.15,
    "formatting": 0.15,
}
```

同步修改 `_REVIEW_PROMPT` 中的维度说明。

### 4.4 修改质量门控阈值

| 阈值 | 位置 | 当前值 |
|------|------|--------|
| 审核通过分 | `reviewer.py:_PASS_THRESHOLD` | 7.0 |
| 审核最大轮次 | `reviewer.py` / `state.py` | 3 |
| 标签去重匹配度 | `organizer.md` | 90% title 相似度 |

## 5. 技术债务与待确认项

| 项目 | 状态 | 说明 |
|------|------|------|
| index.json 维护 | ⚠️ v2 的 step_save 未更新 index.json | 与 AGENTS.md 中 Organizer 定义不一致 |
| v3 的 reviewer 只审核前 5 条 | 📌 有意设计 | 剩余条目未经审核直接进入 organize |

| v2 与 v3 的 model_client 并存 | ⚠️ 代码重复 | 两个版本各有独立的 `model_client.py`，需注意同步更新 |
